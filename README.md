# Лабораторная работа №12: Жизненный цикл Activity (Activity Lifecycle)

## 1. Название проекта

**Grade Clicker** — интерактивное приложение-кликер для набора баллов по курсу "Разработка мобильных приложений".

### Краткое описание:
- Пользователь кликает по изображению и получает +5 баллов за каждый клик
- При достижении пороговых значений (50, 70, 90, 100 баллов) изменяется:
    - Отображаемое изображение
    - Цветовая гамма интерфейса
    - Текстовая оценка и сообщение
- Приложение демонстрирует работу жизненного цикла Activity и сохранение состояния при изменении конфигурации устройства

---

## 2. Цель работы

Изучить жизненный цикл Activity в Android и научиться правильно обрабатывать переходы между состояниями, а также применять механизмы сохранения данных при изменении конфигурации устройства (поворот экрана, смена языка и т.д.).

### В ходе работы было изучено:
- Жизненный цикл Activity и его callback-методы
- Логирование состояния приложения с помощью `Log.d()`
- Проблема потери данных при повороте экрана
- Решение проблемы с помощью `rememberSaveable`
- Разница между `remember` и `rememberSaveable`

## 3. Контрольные вопросы и ответы

### Какие три метода вызываются при запуске приложения?

При первом запуске приложения последовательно вызываются:

1. **`onCreate()`** — Activity создаётся, происходит инициализация UI и переменных
2. **`onStart()`** — Activity становится видимой для пользователя
3. **`onResume()`** — Activity получает фокус и становится активной (готова к взаимодействию)

### Чем отличается onPause() от onStop()?

| Метод | Когда вызывается | Состояние Activity |
|-------|------------------|---------------------|
| `onPause()` | При частичной потере видимости (например, открытие диалогового окна или переход в многозадачность) | Activity частично видима, но не в фокусе |
| `onStop()` | Activity полностью скрыта другим приложением или экраном | Activity не видна пользователю |

**Главное отличие:** `onPause()` вызывается при частичной потере фокуса, `onStop()` — при полной потере видимости.

### Когда вызывается onRestart()?

`onRestart()` вызывается, когда Activity возвращается на передний план после того, как была полностью остановлена (находилась в состоянии `Stopped`). Например:
- Пользователь свернул приложение (нажал Home), а затем вернулся в него
- Пользователь открыл другое приложение, а затем вернулся в наше

Важно: `onRestart()` вызывается **только при возврате из состояния Stop**, но не при первом запуске.

### Что происходит при повороте экрана?

При повороте экрана происходит **изменение конфигурации (configuration change)**:

1. Android уничтожает текущую Activity (`onPause()` → `onStop()` → `onDestroy()`)
2. Создаёт новую Activity с новой конфигурацией (`onCreate()` → `onStart()` → `onResume()`)
3. Без специальных механизмов сохранения все данные теряются

Именно поэтому в приложении использован `rememberSaveable` для сохранения баллов и кликов.

### В чём разница между remember и rememberSaveable?

| Функция | Сохраняет при | Не сохраняет при |
|---------|----------------|-------------------|
| `remember` | Рекомпозиции (перерисовке UI) | Изменении конфигурации (поворот экрана) |
| `rememberSaveable` | Рекомпозиции | — (сохраняет и при повороте) |

**Вывод:** `rememberSaveable` использует `Bundle` для сохранения данных и восстанавливает их при пересоздании Activity.

### Почему onCreate() должен вызывать super.onCreate()?

`super.onCreate()` вызывает реализацию метода в родительском классе (`ComponentActivity`), которая выполняет важную инициализацию:
- Восстановление состояния из `Bundle`
- Инициализацию внутренних компонентов Android
- Настройку lifecycle-компонентов

Без вызова `super.onCreate()` Activity может работать некорректно или вызывать исключения.

### Для чего используется класс Log?

Класс `Log` используется для вывода отладочных сообщений в консоль Logcat. Это помогает разработчику отслеживать выполнение программы, не влияя на пользовательский интерфейс.

**Уровни логирования:**
- `Log.v()` — verbose (подробные сообщения)
- `Log.d()` — debug (отладочные сообщения)
- `Log.i()` — info (информационные сообщения)
- `Log.w()` — warning (предупреждения)
- `Log.e()` — error (ошибки)

### Какой метод вызывается только один раз за время жизни Activity?

**`onCreate()`** вызывается только один раз за весь жизненный цикл Activity. Именно в нём выполняется первичная инициализация: установка UI через `setContent()`, создание объектов, получение ссылок на элементы.
